## 📝 开发规范

### 代码规范
- 使用 ESLint 进行代码检查
- 使用 Prettier 进行代码格式化
- 遵循相应框架的最佳实践

### 提交规范
使用约定式提交格式：
```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

类型示例：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动
